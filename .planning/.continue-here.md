## Bugs Found in Production (non-blocking, need fixing)

### Bug 1: Ack 500 — CAS double-transition
**Location:** `control/internal/lifecycle/lifecycle.go` HandleAck
**Symptom:** `transition state: no rows in result set`
**Root cause:** When agent acks start_sandbox success, control plane tries to transition `starting → running`. But the state was already transitioned on command dispatch. CAS rejects.
**Impact:** container_id and host_port not persisted in sandbox row
**Fix:** Either:
  - Skip transition in ack handler if state is already running
  - Or only transition on the agent event report, not both
**Severity:** MEDIUM — state still correct, but we lose container_id

### Bug 2: Event payload NULL constraint violation
**Location:** `control/internal/lifecycle/lifecycle.go` recordEvent
**Symptom:** `null value in column "payload" of relation "events" violates not-null constraint`
**Root cause:** Ack result body has no payload field, passed as nil to RecordEvent
**Fix:** Default to empty JSON `{}` instead of nil when payload missing
**Severity:** LOW — event logging fails but ack logic continues

### Bug 3: Agent capacity not read from config (FIXED THIS SESSION)
**Was:** agent.go hardcoded CapacityMB=0, caused 400 on register
**Fix applied:** Added CapacityMB + CapacityCPU to Config struct, passed through to register request
**Status:** Deployed and working

### Bug 4: Agent auth used per-node token, server only validates global (FIXED)
**Was:** After register, client stored returned agent_token, used it for polls → 401 loop
**Fix applied:** Added apiToken param to NewClient, authToken() method prefers shared token
**Status:** Deployed and working

### Bug 5: Dockerfile UID 1000 conflict with node:20-slim's existing "node" user
**Fix applied on Server 2:** `userdel -r node 2>/dev/null || true && useradd -u 1000 ...`
**Status:** Server 2 Dockerfile patched in place, needs upstreaming to the source file

### Bug 6: Dockerfile missing WORKDIR for CMD
**Was:** CMD `npx expo start` ran from /app/ but package.json is at /app/app/
**Fix applied on Server 2:** Added WORKDIR /app/app before USER appuser, replaced --non-interactive with CI=1
**Status:** Server 2 patched, needs upstreaming

### Bug 7: Dockerfile uses --production (removes babel-preset-expo)
**Status:** FIXED in plan 01-05-PLAN.md earlier. Image already rebuilt without it.

### Bug 8: Caddy drift detector errors every 60s
**Symptom:** "drift: failed to list Caddy routes: connect: connection refused :2019"
**Root cause:** No Caddy installed yet (Phase 4 config exists but not deployed)
**Severity:** LOW — just log noise, doesn't affect functionality
