SandboxCreate (request body for POST /v1/sandboxes):
```typescript
interface SandboxCreate {
  app_name: string;      // ^[a-z0-9][a-z0-9-]{1,62}$
  user_id: string;
  image: string;         // e.g. 'appx/sandbox:v1'
  resources?: { cpu_cores?: number; memory_mb?: number };
  env?: Record<string, string>;
  idle_timeout_seconds?: number;  // default 1800
  metadata?: Record<string, unknown>;
}
```

Sandbox (response from GET/POST):
```typescript
interface Sandbox {
  id: string;            // UUID
  app_name: string;
  user_id: string;
  node_id: string | null;
  container_id: string | null;
  image: string;
  state: SandboxState;
  url: string;           // https://{app_name}.myappx.live
  resources: { cpu_cores: number; memory_mb: number };
  host_port: number | null;
  created_at: string;    // ISO 8601
  updated_at: string;
  last_active_at: string;
  failure_count: number;
  metadata: Record<string, unknown>;
}
```

SandboxState:
```typescript
type SandboxState = 'pending' | 'starting' | 'running' | 'restarting' | 'destroying' | 'destroyed' | 'failed' | 'stopped';
```

  <behavior>
    RED phase -- write these tests first:

    1. ForgeClient constructor: accepts {baseUrl, apiKey}, stores config
    2. sandboxes.create({app_name, user_id, image}): POST /v1/sandboxes with Bearer token, returns Sandbox
    3. sandboxes.get(id): GET /v1/sandboxes/{id} returns Sandbox
    4. sandboxes.get("app:my-app"): GET /v1/sandboxes/app:my-app returns Sandbox (app_name lookup)
    5. sandboxes.list(): GET /v1/sandboxes returns {sandboxes: Sandbox[]}
    6. sandboxes.list({state: 'running', user_id: 'u1'}): passes query params
    7. sandboxes.destroy(id): DELETE /v1/sandboxes/{id} returns void (202)
    8. sandboxes.restart(id): POST /v1/sandboxes/{id}/restart returns void (202)
    9. sandboxes.pushFiles(id, files): POST /v1/sandboxes/{id}/files, follows 307 redirect, returns FilePushResponse
    10. sandboxes.logs(id): GET /v1/sandboxes/{id}/logs returns string
    11. sandboxes.logs(id, {tail: 50}): passes tail query param
    12. Error handling: 404 throws ForgeNotFoundError, 409 throws ForgeConflictError, 503 throws ForgeServiceError
    13. All requests include Authorization: Bearer {apiKey} header
    14. nodes.list(): GET /v1/nodes returns {nodes: Node[]}
    15. routes.list(): GET /v1/routes returns {routes: Route[]}
    16. healthcheck(): GET /v1/healthz returns {status, postgres, uptime_seconds}
  </behavior>
  <implementation>
    GREEN phase -- implement after tests are written and failing:

    1. **Initialize sdk-ts package**:
       - `npm init -y` with name `@appx/forge-sdk`, main `dist/index.js`, types `dist/index.d.ts`
       - tsconfig.json: target ES2022, module NodeNext, moduleResolution NodeNext, declaration true, outDir dist, strict true
       - jest.config.ts with ts-jest preset
       - Install deps: typescript, @types/node (dev), jest, ts-jest, @types/jest (dev), nock (dev)
       - .gitignore: node_modules, dist, *.tgz

    2. **Create src/types.ts**: Export all interfaces from the OpenAPI spec (listed in interfaces block above). These are hand-written, not generated -- SDK-09 says "generated from OpenAPI" but for 6 types, hand-writing with comments referencing the spec is cleaner and has zero build deps.

    3. **Create src/errors.ts**: Custom error classes extending Error:
       ```typescript
       export class ForgeError extends Error { constructor(public problem: ForgeErrorResponse) {...} }
       export class ForgeNotFoundError extends ForgeError {}
       export class ForgeConflictError extends ForgeError {}
       export class ForgeServiceError extends ForgeError {}
       ```

    4. **Create src/client.ts**: ForgeClient class with:
       ```typescript
       export class ForgeClient {
         constructor(config: { baseUrl: string; apiKey: string });

         sandboxes: {
           create(req: SandboxCreate): Promise<Sandbox>;
           get(idOrApp: string): Promise<Sandbox>;
           list(filters?: { app_name?: string; user_id?: string; state?: SandboxState; node_id?: string; limit?: number }): Promise<Sandbox[]>;
           destroy(id: string): Promise<void>;
           restart(id: string): Promise<void>;
           pushFiles(id: string, files: { path: string; content: string; delete?: boolean }[]): Promise<FilePushResponse>;
           logs(id: string, opts?: { tail?: number; follow?: boolean }): Promise<string>;
         };

         nodes: {
           list(): Promise<Node[]>;
         };

         routes: {
           list(): Promise<Route[]>;
         };

         healthcheck(): Promise<{ status: string; postgres: string; uptime_seconds: number }>;
       }
       ```
       Internal helper: `private async request<T>(method, path, body?, query?): Promise<T>` handles:
       - Base URL + path construction
       - Bearer token header
       - JSON serialization
       - Error response parsing into typed errors
       - 307 redirect following for file push (use `redirect: 'manual'` with fetch, detect 307, re-send to Location header)

       Use native `fetch` (Node 18+ built-in, no axios dependency).

    5. **Create src/index.ts**: Re-export ForgeClient, all types, all errors.

    6. **Write tests in tests/client.test.ts** using nock:
       - nock intercepts HTTP to mock control plane
       - For pushFiles: nock the 307 redirect and the agent endpoint
       - Verify every method sends correct HTTP method, path, headers, query params, body
       - Verify error mapping (404 -> ForgeNotFoundError etc.)
  </implementation>
