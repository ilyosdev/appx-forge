<objective>
Implement the sandbox lifecycle service and HTTP handlers for sandbox CRUD, command acknowledgment,
and container event processing. The lifecycle service orchestrates: create (schedule + dispatch
start_sandbox), destroy (dispatch stop_sandbox), restart (dispatch restart_sandbox), handle ack
(update state on success/failure), handle events (state machine transitions on container events).

Purpose: Core business logic of the control plane -- managing sandbox lifecycle from creation
through running to destruction, with the state machine governing all transitions.

Output: Working sandbox CRUD endpoints and lifecycle service with state machine enforcement.
</objective>

         type Store interface {
             CreateSandbox(ctx, store.CreateSandboxParams) (store.Sandbox, error)
             GetSandbox(ctx, pgtype.UUID) (store.Sandbox, error)
             GetSandboxByAppName(ctx, string) (store.Sandbox, error)

         }

       - `type LifecycleService struct` with store Store, logger *slog.Logger

       - `func New(store Store, logger *slog.Logger) *LifecycleService`

       - `func (ls *LifecycleService) CreateSandbox(ctx, req CreateRequest) (*SandboxResult, error)`:
         a. Generate sandbox UUID
         b. Marshal resources and env to JSON
         c. Call store.CreateSandbox (state is hardcoded "pending" in SQL)
         d. If error contains "unique" or "duplicate" -> return ErrConflict
         e. List healthy nodes via store.ListHealthyNodes
         f. Convert store.Node to scheduler.NodeCandidate
         g. Call scheduler.Schedule(candidates, requiredMB)
         h. If ErrNoNodes/ErrNoCapacity -> return ServiceUnavailable
         i. Call store.AssignSandboxToNode (transitions pending->starting)
         j. Create start_sandbox command with payload: {app_name, image, resources, env}
         k. Record "scheduled" event
         l. Return sandbox with URL computed as https://{app_name}.myappx.live

       - `func (ls *LifecycleService) HandleAck(ctx, cmdID uuid.UUID, sandboxID uuid.UUID, cmdType string, status string, ackResult json.RawMessage) error`:
         a. Map (cmdType, status) to SandboxEvent:
            - (start_sandbox, success) -> EventStarted -> starting->running
            - (start_sandbox, failure) -> EventStartFailed -> starting->failed
            - (stop_sandbox, success) -> EventDestroyed -> destroying->destroyed
            - (restart_sandbox, success) -> no state change (sandbox stays running, agent restarted container)
         b. If state transition needed: call TransitionSandboxState
         c. If start_sandbox success: update sandbox container_id and host_port from ackResult
         d. Record event
         e. Call store.AckCommand with status

       - `func (ls *LifecycleService) HandleEvent(ctx, nodeID uuid.UUID, sandboxID uuid.UUID, eventType string, payload json.RawMessage) error`:
         a. Map agent event_type to SandboxEvent:
            - "container_started" -> EventStarted
            - "container_exited" -> EventContainerExited
            - "container_oom" -> EventContainerExited (same transition)
         b. Get current sandbox state
         c. Call models.NextState(current, event) to validate transition
         d. If valid: call TransitionSandboxState
         e. Record event

       - `func (ls *LifecycleService) DestroySandbox(ctx, sandboxID uuid.UUID) error`:
         a. Get sandbox
         b. Transition to destroying (via state machine: destroy_requested event)
         c. Create stop_sandbox command targeted at sandbox's node_id
         d. Record event

       - `func (ls *LifecycleService) RestartSandbox(ctx, sandboxID uuid.UUID) error`:
         a. Get sandbox (must be in running state)
         b. Create restart_sandbox command targeted at sandbox's node_id
         c. Record event (no state transition -- restart is at Docker level)
