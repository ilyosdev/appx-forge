  <behavior>
    Long-poll:
    - GET /v1/agents/:id/commands?wait=30 with pending commands -> returns immediately with commands array
    - GET /v1/agents/:id/commands?wait=1 with no commands -> returns empty array after ~1s
    - GET /v1/agents/:id/commands with wait > 60 -> clamped to 60
    - GET /v1/agents/:id/commands with no wait param -> defaults to 30s
    - Commands response has format: {"commands": [{id, type, sandbox_id, payload, issued_at, timeout_seconds}]}

    Ack:
    - POST /v1/agents/:id/commands/:cmd_id/ack with {status:"success", result:{}} -> 200
    - POST /v1/agents/:id/commands/:cmd_id/ack with {status:"failure", error:"msg"} -> 200
    - POST /v1/agents/:id/commands/:cmd_id/ack triggers lifecycle.HandleAck

    Event:
    - POST /v1/agents/:id/events with {sandbox_id, event_type, container_id, exit_code, payload} -> 200
    - POST /v1/agents/:id/events triggers lifecycle.HandleEvent
    - POST /v1/agents/:id/events with missing sandbox_id -> 400
  </behavior>
  <action>
    1. Create control/internal/api/agents.go:

       **handlePollCommands:**
       - `func (s *Server) handlePollCommands(w http.ResponseWriter, r *http.Request)`
       - Extract node_id from chi.URLParam(r, "id"), parse UUID
       - Parse "wait" query param (default 30, max 60, min 1)
       - Create context with timeout = wait duration
       - Poll loop:
         a. Call store.PollPendingCommands(ctx, nodeID)
         b. If commands found: marshal to JSON, return immediately
         c. If no commands: sleep 1s (use time.After or context.Done)
         d. If context deadline exceeded: return {"commands": []}
       - Map store.Command to API response format:
         ```go
         type CommandResponse struct {
             ID             string          `json:"id"`
             Type           string          `json:"type"`
             SandboxID      *string         `json:"sandbox_id"`
             Payload        json.RawMessage `json:"payload"`
             IssuedAt       string          `json:"issued_at"`
             TimeoutSeconds int             `json:"timeout_seconds"`
         }
         ```

       **handleAckCommand:**
       - `func (s *Server) handleAckCommand(w http.ResponseWriter, r *http.Request)`
       - Extract node_id and cmd_id from URL params
       - Decode body: {status: "success"|"failure", error?: string, result?: object}
       - Look up the command to get sandbox_id and command_type
       - Call lifecycle.HandleAck(ctx, cmdID, sandboxID, cmdType, status, result)
       - Return 200

       **handleReportEvent:**
       - `func (s *Server) handleReportEvent(w http.ResponseWriter, r *http.Request)`
       - Extract node_id from URL param
       - Decode body: {sandbox_id, event_type, container_id, exit_code, payload}
       - Validate required: sandbox_id, event_type
       - Call lifecycle.HandleEvent(ctx, nodeID, sandboxID, eventType, payload)
       - Return 200

    2. Register routes in server.go authenticated group:
       ```go
       r.Get("/agents/{id}/commands", s.handlePollCommands)
       r.Post("/agents/{id}/commands/{cmd_id}/ack", s.handleAckCommand)
       r.Post("/agents/{id}/events", s.handleReportEvent)
       ```

    3. Create control/internal/api/agents_test.go:
       - Mock store and lifecycle service
       - **Long-poll with commands:** seed mock with pending commands -> returns immediately
       - **Long-poll timeout:** mock returns empty -> set wait=1 -> returns empty array after ~1s
       - **Long-poll max wait clamped:** wait=120 -> clamped to 60
       - **Ack success:** POST with {status:"success"} -> 200, lifecycle.HandleAck called
       - **Ack failure:** POST with {status:"failure", error:"msg"} -> 200, HandleAck called
       - **Event report:** POST with valid body -> 200, lifecycle.HandleEvent called
       - **Event missing sandbox_id:** -> 400

    4. The long-poll test with wait=1 should complete quickly (~1-2s). Use a short wait value
       in tests to avoid slow tests. Verify the handler respects the context deadline.
  </action>

    - Long-poll returns empty array on timeout

    - Event handler records event and triggers lifecycle

       - `func (s *Server) handleFilePush(w http.ResponseWriter, r *http.Request)`
       - Extract sandbox {id} from URL param, parse UUID (or app:name lookup)
       - Get sandbox from store
       - If not found: 404
       - If sandbox has no node_id (not yet assigned): 503 "sandbox not yet scheduled"

       - Return 307 Temporary Redirect

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent -> long-poll | Agent polls with agent_token |
| SDK -> file push | SDK pushes files, gets redirect to agent |
| control -> agent redirect | Signed URL prevents unauthorized file push to agent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Spoofing | file push redirect | mitigate | HMAC-signed URL with 60s expiry; only control plane can sign (knows HMAC secret) |
| T-03-15 | Tampering | file push URL | mitigate | HMAC covers full URL path+query; any modification invalidates signature |
| T-03-16 | Denial of Service | long-poll | mitigate | wait param capped at 60s; context timeout prevents goroutine leak |
| T-03-17 | Information Disclosure | command payload | accept | Commands returned to requesting agent only (filtered by node_id); payloads contain sandbox config (not secrets) |
</threat_model>
