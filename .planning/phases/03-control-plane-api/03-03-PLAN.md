<objective>
Implement node registration and heartbeat handlers on the control plane server. Registration is
unauthenticated (it provides the agent token). Heartbeat is authenticated and updates node
resource metrics. Re-registration with same hostname+IP is idempotent.

Purpose: Enables agents to register and maintain liveness, which is prerequisite for command
dispatch and sandbox scheduling.

Output: Working registration and heartbeat endpoints with httptest coverage.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent -> registration | Unauthenticated endpoint, agent proves identity by hostname+IP |
| agent -> heartbeat | Authenticated via agent_token (Bearer) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Spoofing | node registration | mitigate | Re-registration overwrites token (old token invalidated); Tailscale mesh limits access to trusted nodes |
| T-03-07 | Spoofing | heartbeat | mitigate | Bearer token required; token is per-node, scoped |
| T-03-08 | Denial of Service | registration | accept | Registration is unauthenticated but creates at most 1 row per hostname+IP pair; Tailscale mesh limits exposure |
| T-03-09 | Information Disclosure | agent_token | mitigate | Token generated with crypto/rand, stored in DB, returned only during registration, never logged |
</threat_model>
