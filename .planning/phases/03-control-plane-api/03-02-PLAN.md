<feature>
  <name>Bin-packing scheduler</name>
  <files>control/internal/scheduler/scheduler.go, control/internal/scheduler/scheduler_test.go</files>
  <behavior>
    Inputs: []NodeCandidate (id, capacity_mb, used_mb, status), requiredMB int
    Output: selected node UUID, or error

    Cases:
    - 3 healthy nodes with free RAM [2000, 5000, 3000], request 512MB -> picks node with 5000 free (most free)
    - 2 healthy + 1 draining, draining has most free -> picks healthy with most free, not draining
    - 2 healthy + 1 unhealthy, unhealthy has most free -> picks healthy with most free, not unhealthy
    - All nodes healthy but none has enough free RAM for request -> ErrNoCapacity
    - No nodes at all (empty slice) -> ErrNoNodes
    - 1 healthy node with exactly enough free RAM -> picks that node (edge case: exact fit)
    - 2 healthy nodes with same free RAM -> picks first (deterministic tiebreak)
    - Node with status "removed" is excluded
  </behavior>
  <implementation>
    1. Define NodeCandidate struct with plain Go types (UUID, int32, string) -- no pgtype dependency
    2. Define ErrNoNodes and ErrNoCapacity sentinel errors
    3. func Schedule(candidates []NodeCandidate, requiredMB int32) (uuid.UUID, error)
       - Filter: keep only status == "healthy"
       - Filter: keep only (capacity_mb - used_mb) >= requiredMB
       - Sort by free RAM descending (capacity_mb - used_mb)
       - Return first candidate's ID
       - If no candidates after filtering: return appropriate error
    4. Keep it simple -- no interface, no struct, just a function. The caller (lifecycle service)
       converts store.Node to NodeCandidate before calling.
  </implementation>
</feature>
