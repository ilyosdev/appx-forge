<objective>
TDD implementation of two background goroutines: idle sandbox reaper and Caddy routing drift detector.

Purpose: Idle reaper reclaims resources by stopping sandboxes that have been inactive beyond their configured timeout (default 30min). Drift detector ensures Caddy routing state matches Postgres truth by fixing discrepancies every 60s -- this is the safety net for any route add/remove that failed during normal lifecycle operations.

Output: IdleReaper and DriftDetector types with Run(ctx) methods, ready to be started as goroutines in main.go (wired in Plan 03).

**Step 2: GREEN -- Implement drift.go:**

Create `control/internal/routing/drift.go`:

```go
package routing

// DriftDetector periodically compares Caddy routes with Postgres sandbox state
// and fixes discrepancies.
type DriftDetector struct {
    proxy    RouteListFetcher
    store    DriftStore
    logger   *slog.Logger
    interval time.Duration // default 60s
}
```

- `NewDriftDetector(proxy, store, logger, interval)` constructor
- `Run(ctx context.Context)` -- ticker loop, calls `detect()` on each tick
- `detect(ctx context.Context)`:
  1. Call `proxy.ListRoutes()` -> build map[appName]Route
  2. Call `store.ListSandboxesByState("running")` -> build map[appName]Sandbox
  3. For each Caddy route NOT in Postgres running map: `proxy.RemoveRoute(appName)` (stale route)
  4. For each Postgres running sandbox NOT in Caddy map: look up node via `store.GetNode(nodeID)`, build upstream `tailscaleIP:hostPort`, call `proxy.AddRoute(Route{AppName, SandboxID, Upstream})` (missing route)
  5. Log summary: "drift check complete: added=%d removed=%d"
  6. If either ListRoutes or ListSandboxesByState fails, log error and return early (don't fix partial data)

Run `go test ./control/internal/routing/ -run TestDrift -count=1 -race` to confirm GREEN.

| postgres->control | Idle reaper reads sandbox state from database |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-05-03 | D (Denial of Service) | idle_reaper.go | mitigate | Per-sandbox idle_timeout_seconds prevents blanket reap; continues on individual failures |
| T-05-04 | T (Tampering) | drift.go | mitigate | Drift detector only adds/removes routes -- never modifies containers. Routes only, never sandbox state. Postgres is source of truth. |
| T-05-05 | I (Information Disclosure) | drift.go | accept | Route list from Caddy contains app names (public subdomain info, not secrets) |
